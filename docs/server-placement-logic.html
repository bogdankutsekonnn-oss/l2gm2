L2GM — Логика размещения серверов

Техническая документация | Версия 1.0 | Февраль 2026

1. Структура данных сервера

{ "id": 1, "name": "SERVER NAME", "url": "https://example.com", "chronicle": "High Five", "rate": 100, "startDate": "2026-02-27", // дата открытия сервера "cardType": "vip", // тип размещения "icons": ["hot-start"], // платные иконки "avatarUrl": null, "ownerId": null, "description": null, "createdAt": "2026-02-01", // дата создания заявки "expiresAt": "2026-03-03" // дата истечения (createdAt + 30 дней) }

2. Типы размещения (тарифы)

Тип Приоритет Стоимость / 30 дней Описание
Premium 1 (высший) $40 за колонку Яркий дизайн, самый верх списка, дублирование в блоках
VIP 3 $20 за колонку Оригинальный дизайн, отдельный блок, выше обычных
TOP 4 $9 ($5 эконом) Стандартный VIP, выше basic, заметное оформление
Basic 5 (низший) Бесплатно Обычная карточка, требуется кнопка на сайте сервера

3. Срок размещения и экспирация

Правила расчёта

  • Платные серверы (premium, vip, top): expiresAt = createdAt + 30 дней
  • Бесплатные серверы (basic): expiresAt = null — бессрочно
  • Отсчёт 30 дней идёт от createdAt (дата одобрения заявки), а НЕ от startDate

Что происходит после истечения 30 дней

1
expiresAt < сегодня — система определяет, что размещение истекло
2
Сервер получает effectiveType = "basic" (в JSON cardType не меняется)
3
Сервер теряет: приоритетную сортировку, попадание в платные секции, визуальное оформление
4
Сервер продолжает отображаться как обычная (basic) карточка в общих категориях
Важно: экспирация проверяется в реальном времени при каждом рендере. Нет cron-задач или фоновых процессов — логика полностью клиентская.

4. Отображение на главной странице

Серверы распределяются по двум колонкам на основе startDate относительно текущей даты (МСК, UTC+3):

Левая колонка — будущие серверы

Категория Условие Кто попадает
Скоро откроются startDate ≥ сегодня Только платные (premium / vip / top)
Завтра startDate = завтра Все серверы
Ближайшие 7 дней startDate через 2–7 дней Все серверы
Через неделю и более startDate > 7 дней Все серверы

Правая колонка — прошлые серверы

Категория Условие Кто попадает
Уже открылись startDate < сегодня Только платные (premium / vip / top)
Предыдущие 7 дней startDate 1–7 дней назад Все серверы
Неделю назад и более startDate > 7 дней назад Все серверы
Лимиты платных секций: максимум 1 Premium + 10 остальных в секциях «Скоро откроются» и «Уже открылись».

5. Миграция сервера по колонкам

Пример: сервер добавлен 1 февраля, startDate = 27 февраля, cardType = vip, expiresAt = 3 марта

Период Где отображается Статус
1–19 фев Левая: «Скоро откроются» + «Через неделю и более» VIP
20–25 фев Левая: «Скоро откроются» + «Ближайшие 7 дней» VIP
26 фев Левая: «Скоро откроются» + «Завтра» VIP
27 фев Левая: «Скоро откроются» VIP (день открытия)
28 фев – 2 мар Правая: «Уже открылись» + «Предыдущие 7 дней» VIP
3 марта Правая: только «Предыдущие 7 дней» Basic (истёк!)
7+ марта Правая: «Неделю назад и более» Basic

6. Сортировка серверов

Внутри каждой категории серверы сортируются в два этапа:

  1. По типу карточки (с учётом экспирации): Premium → VIP → Top → Basic
  2. По дате старта (при одинаковом типе): ближайшие сначала
Экспирация в сортировке: если expiresAt прошёл, сервер сортируется с приоритетом Basic (5), даже если в JSON стоит cardType: "vip".

7. Форма добавления сервера

При отправке формы (/add-server) формируется объект:

{ name, url, chronicle, rate, startDate, cardType: "vip", // из выбранного тарифа icons: ["hot-start"], // из выбранных допов createdAt: "2026-02-28", // текущая дата (автоматически) expiresAt: "2026-03-30", // createdAt + 30 (автоматически) email, contacts, serverTypes // для модерации }

9. Особенности и ограничения

  • Все даты сравниваются в часовом поясе Москва (UTC+3)
  • Серверы с startDate = сегодня и cardType = basic не попадают ни в одну категорию (known gap)
  • Один платный сервер может отображаться в двух категориях одновременно (например, «Скоро откроются» + «Ближайшие 7 дней»)
  • cardType в JSON не меняется при экспирации — меняется только effectiveType при рендере
  • Данные пока хранятся в JSON, подготовлено к миграции на Supabase
